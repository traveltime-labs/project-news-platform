# 全球新聞 AI 監測與個人化閱覽系統
## 開發企劃書 V1.4

---

## 1. 專案目標

建立一個整合自動化爬蟲、AI 處理與個人化管理的新聞監測平台。透過 Google News RSS 關鍵字訂閱，針對特定高價值領域（科技、經濟、世界局勢、外星人研究及台灣在地新聞）進行全球追蹤，並透過 Telegram 提供定時彙整快報。使用者可在系統內直接閱讀全文，無廣告干擾。

---

## 2. 技術架構

### 2-1. 技術總覽

| 模組 | 技術選型 | 說明 |
|------|---------|------|
| 前端展示 | Vite + Vue 3 + Pinia + Tailwind CSS | 部署於 Vercel（免費） |
| 爬蟲引擎 | Python + requests + feedparser | 純 RSS，不需要 Playwright |
| 爬蟲排程 | GitHub Actions | 定時觸發，免費額度內（每月 2000 分鐘） |
| 資料儲存 | Google Sheets API | 輕量化資料庫，後續可遷移至 PostgreSQL |
| 翻譯 API | OpenAI GPT-4o-mini（按需） | 由 Vercel Serverless Function 呼叫 |
| 全文萃取 | @mozilla/readability + readability.css | 即時 fetch 原文，去除廣告後顯示正文 |
| 通知中心 | Telegram Bot API | 定時快報 + 異常告警 |

### 2-2. 爬蟲來源策略

> 原則：以 Google News RSS 關鍵字訂閱為主，特殊來源保留獨立 RSS。

**Google News RSS（關鍵字驅動）：**

| 類別 | RSS 網址 |
|------|---------|
| 國際科技 | `news.google.com/rss/headlines/section/topic/TECHNOLOGY?hl=en-US&gl=US&ceid=US:en` |
| 國際財經 | `news.google.com/rss/headlines/section/topic/BUSINESS?hl=en-US&gl=US&ceid=US:en` |
| 世界局勢 | `news.google.com/rss/headlines/section/topic/WORLD?hl=en-US&gl=US&ceid=US:en` |
| 台灣新聞 | `news.google.com/rss/headlines/section/geo/Taiwan?hl=zh-TW&gl=TW&ceid=TW:zh-Hant` |
| UFO／外星人 | `news.google.com/rss/search?q=UFO+UAP&hl=en-US&gl=US&ceid=US:en` |
| 自訂關鍵字 | `news.google.com/rss/search?q=關鍵字&hl=en-US&gl=US&ceid=US:en` |

**獨立 RSS（保留）：**

| 來源 | 類別 | RSS 網址 |
|------|------|---------|
| Reddit r/UFOs | 外星人 | `reddit.com/r/UFOs/top/.rss?sort=top&t=day&limit=20` |
| NASA | 外星人/科學 | `nasa.gov/news-release/feed/` |

> 💡 **關鍵字可在控制面板動態新增／修改／刪除，不需要改程式碼。**  
> ⚠️ **Playwright 已完全移除**，所有來源均使用 RSS，架構更簡單穩定。

## 3. 核心功能規格

### 3-1. 多來源爬蟲模組

**穩定性機制：**
- 心跳檢測：若單一來源連續三次抓取結果為空，自動發送 Telegram 異常告警（狀態存於 Google Sheets `Settings` 工作表）
- 批量寫入：爬蟲抓取完成後一次性寫入 Google Sheets，避免觸發 API Quota（429 Error）
- 去重邏輯：`News` 工作表新增 `url_hash` 欄位（URL 的 MD5 雜湊值），寫入前比對是否已存在
- Google 跳轉連結解析：Google News RSS 的連結為跳轉網址，爬蟲需解析出真正的原文網址後再儲存

### 3-2. 資料儲存結構（Google Sheets）

| 工作表 | 欄位 | 說明 |
|--------|------|------|
| `News` | 時間 / 來源 / 標題 / URL / url_hash / 摘要 / 分類 / 翻譯標題 / 翻譯摘要 / 已發送 | 不儲存全文，全文由 Serverless Function 即時萃取 |
| `Bookmarks` | 新聞 ID / 收藏時間 | 使用者點擊愛心即時同步 |
| `Settings` | 來源名稱 / RSS網址 / 類別 / 啟用狀態 / 心跳失敗次數 / 通知排程時間 | 支援動態新增關鍵字來源 |

### 3-3. 個人化閱覽與控制台（Vue 3）

- **新聞牆介面：** 卡片式佈局，支援類別篩選與愛心收藏
- **即時搜尋：** 輸入關鍵字後 debounce 300ms 觸發，模糊比對標題與摘要（原文＋譯文），可與類別篩選同時並存
- **閱讀全文：** 點擊「閱讀全文」觸發 `/api/article`，Serverless Function 即時 fetch 原文 HTML，經 Readability.js 萃取正文後，以 Modal 呈現，套用 readability.css 排版，支援字體大小調整
- **按需翻譯：** 點擊按鈕觸發 Vercel Serverless Function 呼叫 GPT-4o-mini，翻譯結果回寫至 Sheets
- **控制面板：** 動態管理 RSS 來源（新增／修改／刪除關鍵字）、排程軟開關

### 3-4. 通知系統

- 定時快報：彙整摘要 + 附帶「查看全部」連結（導回 Vercel 前端），避免超過 Telegram 4096 字元上限
- 即時告警：系統異常（爬蟲失效）時立即傳送告警

## 4. 系統工作流程

```
0. 初始化   setup_sheets.py（僅執行一次）
            └─ 自動建立 News / Bookmarks / Settings 工作表與欄位
               並寫入預設來源清單

1. 觸發     GitHub Actions（cron）
            └─ 定時啟動 Python 爬蟲腳本

2. 檢索     Python 爬蟲
            ├─ 讀取 Settings 取得啟用的 RSS 來源清單
            ├─ feedparser 抓取各 RSS feed
            └─ 解析 Google News 跳轉連結，取得原文網址

3. 去重     Python 爬蟲
            └─ 比對 url_hash，過濾已存在資料

4. 儲存     Python 爬蟲
            └─ 整批寫入 News 工作表（不含全文）

5. 彙整     GitHub Actions（cron）
            └─ 到達指定時間，撈取未發送新聞 → 格式化 → 發送 Telegram

6. 互動     Vercel 前端 + Serverless Function
            ├─ 使用者瀏覽、類別篩選、即時搜尋
            ├─ 點擊「閱讀全文」→ /api/article 即時萃取正文 → Modal 顯示
            └─ 點擊翻譯或收藏，透過 API 更新 Sheets
```

## 6. 風險評估

| 風險 | 說明 | 對策 |
|------|------|------|
| Google News RSS 演算法過濾 | 部分新聞可能被 Google 認為不重要而不出現 | 保留 Reddit、NASA 獨立 RSS 補充特定來源 |
| Google 跳轉連結解析失敗 | 少數連結可能無法解析出原文網址 | 解析失敗時保留 Google 跳轉網址，仍可正常開啟 |
| Google Sheets API 限制 | 每分鐘請求配額 | 批量寫入（append_rows），避免 429 Error |
| 搜尋效能 | Serverless Function 需將整張表撈入記憶體過濾 | 幾千筆以內無虞；遷移 PostgreSQL 後改為 `ILIKE` 查詢 |
| 全文萃取失敗 | 付費牆（NYT、WSJ 等）Readability.js 無法萃取 | 前端顯示「無法取得全文」提示，附原文連結作為備援 |
| Vercel Serverless 時限 | 免費方案 10 秒上限 | 翻譯與搜尋通常 3 秒內完成；全文萃取視來源速度，一般 5 秒內 |
| GitHub Actions 時限 | 免費 2000 分鐘/月 | 每次約 3 分鐘（純 RSS，無 Playwright），一天兩次 = 180 分鐘/月 |
