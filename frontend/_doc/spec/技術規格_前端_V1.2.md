## 3. API 介面定義

### 3-1. 全文萃取 API（api/article.ts）（新增）

#### `fetchArticle(url)`

```typescript
interface FetchArticleResponse {
  data: ArticleContent | null  // null 表示萃取失敗（付費牆等）
  error?: string
}

async function fetchArticle(url: string): Promise<FetchArticleResponse> {
  const res = await fetch(
    `${import.meta.env.VITE_API_BASE_URL}/article?url=${encodeURIComponent(url)}`
  )
  if (!res.ok) throw new Error('fetch failed')
  return res.json()
}
```

**Endpoint：** `GET /api/article?url=https://techcrunch.com/xxx`

**錯誤處理：**

| HTTP 狀態碼 | 情境 | 前端處理 |
|------------|------|---------|
| 200 + data: null | 付費牆或動態頁面，無法萃取 | Modal 顯示「無法取得全文」+ 原文連結按鈕 |
| 408 | fetch 原文逾時 | Modal 顯示「載入逾時」+ 原文連結按鈕 |
| 500 | 伺服器錯誤 | Modal 顯示「發生錯誤」+ 原文連結按鈕 |

## 4. Pinia Store 定義

### 4-1. news.ts

```typescript
export const useNewsStore = defineStore('news', () => {
  const items = ref<NewsItem[]>([])
  const total = ref(0)
  const isLoading = ref(false)
  const activeCategory = ref<NewsCategory | null>(null)
  const searchQuery = ref('')
  const offset = ref(0)
  const LIMIT = 20

  async function loadNews(reset = false) {
    if (reset) offset.value = 0
    isLoading.value = true
    const res = await fetchNews({
      category: activeCategory.value ?? undefined,
      q: searchQuery.value || undefined,
      limit: LIMIT,
      offset: offset.value
    })
    items.value = reset ? res.data : [...items.value, ...res.data]
    total.value = res.total
    offset.value += LIMIT
    isLoading.value = false
  }

  function setCategory(category: NewsCategory | null) {
    activeCategory.value = category
    loadNews(true)
  }

  function setSearchQuery(q: string) {
    searchQuery.value = q
    loadNews(true)
  }

  function applyTranslation(newsId: string, titleZh: string, summaryZh: string) {
    const item = items.value.find(n => n.id === newsId)
    if (item) {
      item.titleZh = titleZh
      item.summaryZh = summaryZh
    }
  }

  return {
    items, total, isLoading, activeCategory, searchQuery,
    loadNews, setCategory, setSearchQuery, applyTranslation
  }
})
```

### 4-2. bookmarks.ts（同 V1.1）

```typescript
export const useBookmarksStore = defineStore('bookmarks', () => {
  const bookmarkedIds = ref<Set<string>>(new Set())

  async function loadBookmarks() {
    const res = await fetchBookmarks()
    bookmarkedIds.value = new Set(res.data.map(b => b.newsId))
  }

  async function toggle(newsId: string) {
    if (bookmarkedIds.value.has(newsId)) {
      bookmarkedIds.value.delete(newsId)
      await removeBookmark(newsId)
    } else {
      bookmarkedIds.value.add(newsId)
      await addBookmark(newsId)
    }
  }

  function isBookmarked(newsId: string) {
    return bookmarkedIds.value.has(newsId)
  }

  return { bookmarkedIds, loadBookmarks, toggle, isBookmarked }
})
```

## 5. 元件規格

### 5-1. ArticleModal.vue（新增）

**Props：**

| Prop | 型別 | 必填 | 說明 |
|------|------|------|------|
| `news` | `NewsItem \| null` | ✅ | null 時 Modal 關閉 |

**內部狀態：**

| 狀態 | 型別 | 說明 |
|------|------|------|
| `article` | `ArticleContent \| null` | 萃取後的全文資料 |
| `isLoading` | `boolean` | 正在 fetch 全文 |
| `error` | `string \| null` | 錯誤訊息 |
| `fontSize` | `number` | 字體大小（預設 18），單位 px |

**行為：**
- `news` prop 有值時自動呼叫 `fetchArticle(news.url)`
- 全文以 `v-html` 渲染，套用 `readability.css` 的樣式 scope
- 字體大小調整：`+` / `-` 按鈕控制 `fontSize`，範圍 14～24px
- 萃取失敗時顯示備援區塊（見下方）
- 按 `Esc` 或點擊遮罩關閉 Modal

**備援顯示（萃取失敗）：**
```
無法取得全文
可能原因：付費牆或動態網頁限制

[前往原文閱讀 →]
```

**樣式結構：**
```vue
<template>
  <Teleport to="body">
    <div class="modal-overlay" @click.self="$emit('close')">
      <div class="modal-container">

        <!-- Header -->
        <div class="modal-header">
          <h2>{{ article?.title ?? news?.title }}</h2>
          <div class="controls">
            <button @click="fontSize = Math.max(14, fontSize - 2)">A-</button>
            <button @click="fontSize = Math.min(24, fontSize + 2)">A+</button>
            <a :href="news?.url" target="_blank">原文連結</a>
            <button @click="$emit('close')">✕</button>
          </div>
        </div>

        <!-- 內容區 -->
        <div
          class="reader-content"
          :style="{ fontSize: `${fontSize}px` }"
          v-html="article?.content"
        />

      </div>
    </div>
  </Teleport>
</template>
```

### 5-4. SearchBar.vue（同 V1.1）

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { useDebounceFn } from '@vueuse/core'
import { useNewsStore } from '@/stores/news'

const newsStore = useNewsStore()
const inputValue = ref('')

const debouncedSearch = useDebounceFn((q: string) => {
  newsStore.setSearchQuery(q)
}, 300)

function onInput(e: Event) {
  const q = (e.target as HTMLInputElement).value
  inputValue.value = q
  if (q === '') {
    newsStore.setSearchQuery('')
  } else {
    debouncedSearch(q)
  }
}
</script>
```

## 7. 錯誤處理總覽

| 情境 | 處理方式 |
|------|---------|
| 初始載入失敗 | 顯示錯誤訊息區塊 + 重試按鈕 |
| 載入更多失敗 | Toast 提示 + 保留現有資料 |
| 搜尋無結果 | 顯示「找不到相關新聞」空狀態畫面 |
| 搜尋請求失敗 | Toast 提示 + 保留上次結果 |
| 全文萃取失敗／付費牆 | Modal 內顯示備援區塊 + 原文連結按鈕 |
| 全文載入逾時 | Modal 內顯示「載入逾時」+ 原文連結按鈕 |
| 翻譯失敗 | Toast 提示 + 按鈕恢復可點擊 |
| 收藏寫入失敗 | Toast 提示 + UI 回滾（取消愛心狀態） |
| 設定儲存失敗 | Toast 提示 + 保留上次成功的設定值 |
